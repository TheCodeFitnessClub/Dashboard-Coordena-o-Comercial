Reunião diária (10–15 min) · Foco: execução

Output esperado: prioridades e bloqueios resolvidos. Adiciona notas a falar e marca o check quando for discutido.
  • 1. Performance dia anterior: contactos, conversões, incidentes críticos.
  • 2. Estado da operação: riscos identificados e como atacá-los.
  • 3. Prioridades do dia: 3 máximas, com ações.
  • 4. Bloqueios: problemas, dependências, decisões.
Disciplina: dados preparados antes · termina com ação definida.

Reunião semanal · Foco: correção

Output obrigatório: plano de correção semanal formalizado + ações atribuídas.
  • Análise de KPIs: Real vs objetivo + tendência vs semana anterior.
  • Revisão de pipeline.
  • Identificação de desvios.
  • Plano de ação corretivo.
Nenhum KPI é analisado sem comparação temporal · Nenhum desvio fica sem plano de ação.

Log das reuniões semanais

Reunião mensal · Foco: evolução

Output obrigatório: relatório de performance + decisões + plano do mês seguinte.
  • Performance global e tendências.
  • Avaliação de equipa: performance individual, consistência, formação.
  • Ajustes operacionais e definição de melhorias.
Diário → execução · Semanal → correção · Mensal → evolução.

Backup & Reset

Os teus dados estão guardados localmente no navegador (localStorage). Não saem do teu computador. Se mudares de máquina ou navegador, exporta o backup primeiro.

Exportar XLSX gera um workbook com todas as abas para partilhar com a Direção ou ter como cópia de segurança.

Backup JSON é o método recomendado para clonar o estado para outra máquina.
